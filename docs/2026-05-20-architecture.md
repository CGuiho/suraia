# 🌻 GUIHO Suraia — Architecture Specification (AI-First UI Blueprint)

**Date:** 2026-05-20  
**Status:** Approved  

---

## 1. Vision & Core Philosophy

**GUIHO Suraia** (`@guiho/suraira`) is an AI-first UI blueprint library. It is designed to store neutral component structures, behavior logic, style tokens, and translation instructions that AI coding assistants use to generate custom components directly inside a target application.

Instead of distributing pre-built framework components (e.g., React, Vue, or Svelte libraries), Suraia provides a universal baseline of component specifications, accessibility contracts, and interaction controllers.

> **Key Rule:** Build the component knowledge once. Let AI compile/generate it everywhere.

---

## 2. Consumption Model (How it is Used)

### 2.1 Developer-Facing Lifecycle
Suraia is designed specifically for **AI-augmented development workflows**:

```txt
┌────────────────────────────────────────────────────────┐
│                   GUIHO Suraia                        │
│   (devDependency in target project's package.json)    │
└──────────────────────────┬─────────────────────────────┘
                           │
                           │ AI assistant reads:
                           │  - Component Blueprint (JSON)
                           │  - Behavior Logic (TS)
                           │  - Styling Spec (CSS)
                           │  - Framework Guidelines (MD)
                           ▼
┌────────────────────────────────────────────────────────┐
│                   Target Application                   │
│  - React / Vue / Svelte Component generated by AI     │
│  - CSS classes generated and styled inside project    │
└────────────────────────────────────────────────────────┘
```

1. **DevDependency-Only:** The package `@guiho/suraira` is installed only as a `devDependency` in the developer's project.
2. **Zero Production Runtime Footprint:** The library is never imported at runtime in production.
3. **Local Code Generation:** All components, HTML anatomy, behavior logic, and CSS style sheets are compiled, generated, or copied directly into the target project's codebase. The project gains full ownership and customization of the generated code.

## 3. System Layers

Suraia is split into four primary layers:

```txt
┌────────────────────────────────────────────────────────┐
│                     1. Themes                          │
│   (JSON design tokens & scoped CSS variables)          │
└──────────────────────────┬─────────────────────────────┘
                           ▼
┌────────────────────────────────────────────────────────┐
│                  2. Component Specs                    │
│   (Anatomy, states, variants & markup blueprints)      │
└──────────────────────────┬─────────────────────────────┘
                           ▼
┌────────────────────────────────────────────────────────┐
│                 3. Behavior Controllers                 │
│   (Framework-agnostic interaction JS/TS engines)       │
└──────────────────────────┬─────────────────────────────┘
                           ▼
┌────────────────────────────────────────────────────────┐
│                   4. Prompt Guides                     │
│   (Markdown templates for AI code generation)          │
└────────────────────────────────────────────────────────┘
```

### 3.1 Themes & Design Tokens
* Standardized CSS Custom Properties (e.g., `--sui-color-primary`, `--sui-radius-md`).
* Scoped under profile data attributes (e.g., `[data-sui-profile="apple-like"]`) to allow dynamic swapping of visual languages.
* Exposed as JSON files for machine readability by AI engines.

### 3.2 Component Specifications (Blueprints)
Each component contains:
* **`[name].json`:** Machine-readable metadata describing components, slots, variants, states, and accessibility bindings.
* **`[name].css`:** Pure, structural, layout-only CSS (free of visual theme choices like specific colors or shadows).
* **`[name].structure.html`:** Reference semantic markup mapping out structure, classes (`.sui-button`), and attributes (`data-sui-variant`).

### 3.3 Behavior Controllers
* Pure, framework-independent JavaScript/TypeScript files (e.g., `button.ts`).
* Defines the exact state machine and interaction handling logic (e.g., handling click, disabling buttons, running spinner during loading states).
* Reusable directly in vanilla setups or serves as the strict reference implementation for framework wrappers.

### 3.4 Prompt Guides
* Standardized instructions (`react.md`, `vue.md`, `svelte.md`) that teach an AI agent how to read the blueprint metadata and behavior script to generate idiomatic, clean code for the target framework.

## 4. Initial MVP Component Set

The baseline specification starts with five essential components:

| Component | Focus Area |
| :--- | :--- |
| **Button** | Variant mappings, states, basic click/disabled/loading behavior, ARIA roles. |
| **Input** | Validation, labels, error states, and browser form behavior. |
| **Card** | Layout patterns, structural slots, static styling boundaries. |
| **Badge** | Static style indicators for tokens, tones, and size variants. |
| **Dialog** | Keyboard trapping, overlay accessibility, portal patterns. |
